# COCO 客户工作台 v10

打开 `app.html` 即可使用；`index.html` 是默认入口，`workbench.html` 保留旧版资料看板。

- 数据来源：COCO Notion 导出 zip 中筛选后的业务表格。
- 已排除：团队成员资料、账号列表、聊天记录存档，以及用户另外提供的人事/人格/客户工作原则类文档。
- v10 定位：一页白板优先回答本周做了什么、哪些已完成、哪些未闭环、下周要确认什么。
- 核心逻辑：一次录入，三处呈现——工作动态流、数据中心、白板自动稿。
- 白板结构：本周可汇报成果、客户心理变化、下次会必须问清楚、微信群闭环、客户最新要求、上次 Action、本周交付、数据对比。
- 会议分析：外层只列每次会议重点；点开后查看完整分析、Action 和原文位置。
- 会议档案：展示完整会议分析与 Action；转录原文只显示内部语料状态，不直接渲染。
- 社媒成果：已发布且表内有 URL 的内容显示可点击链接。
- 飞书社媒看板：https://my.feishu.cn/wiki/KGJJwc6iOiFMeokvIJrcQM81n1d?sheet=1Jbfs5
- 录入同步：页面会优先提交到 `/api/entries`；GitHub Pages 下无后端，会本地保存；迁到 Vercel 并配置环境变量后即可自动写回仓库并全员同步。

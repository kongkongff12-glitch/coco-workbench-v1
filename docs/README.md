# COCO 客户白板 v2

打开 `app.html` 即可使用；`index.html`、`workbench.html`、`docs/` 下入口均同步为同一版。

## 这版的定位

这个系统不是“上次待办回看”，也不是“本周期完成清单”。它的核心用途是：

- 替代每次客户会议前人工做白板。
- 让老板和全员快速看懂客户当前态、每周会议主线、卡点、客户心理变化。
- 每一周白板独立保存，点左侧周时间线切换，不互相覆盖。

## 页面结构

1. 顶部当前态，跨周不变：KPI 概览条、SEO 阶段进度条、看完整甘特入口。
2. 左侧周时间线：会议日期、会议名、一句话摘要、红灯数；最新周在最上。
3. 右侧周白板：默认展示最新一周，点哪周就切到哪周。

## 单周白板顺序

1. 本周重点（结论）钉在最上，用大字写主线和关键卡点。
2. 会议议程卡打头。
3. 一个话题一张卡，按议程和重要度排序；待讨论、卡点话题优先。
4. 系统补充卡放最后：客户开口的事、客户动向、下次会必须问清楚、下周计划。

## 卡片规则

每张卡统一为：红色话题标题、右上角负责人、总结、实物。

- 议程：编号列表，每项带时长。
- 表格：列由数据决定，不写死。
- 进度甘特：已完成=墨色实条，进行中=红条，计划=蓝色虚线条，今天=黄色竖线，任务前标 P0/P1/P2。
- 问题汇总：分已解决（绿）和待修复（红），每条带 P0/P1/P2。
- 社媒矩阵：平台 × 周表格，浏览/播放/阅读量高亮，底部挂飞书源链接。
- 链接组：公众号、推特、Notion、PDF 等链接小标签。
- 数据指标：上期 → 本期（±%）+ 迷你趋势线；本周没更新灰显。

## 全局规则

- 结论和重点永远在最上，证据和细节往下折叠。
- 交付、话题卡、数据、卡点显示负责人；下周计划不显示负责人。
- 客户视图隐藏负责人、内部卡点、会议原文截图、全客户总览、录入、名单、物料。
- 已发布内容显示平台链接；未发布显示“待发布”；没有链接不显示。
- 零交付必须显式红色显示“本周无交付”。
- 数据必须带对比和趋势，孤立数字不合格。
- 状态只用颜色提醒：待对齐/待贵方提供=红，已回应=绿，处理中=黄。
- 待对齐、待贵方提供、逾期、未更新必须显式呈现，不折叠隐藏。
- 飞书社媒、Notion 各表作为镜像展示，保留“看飞书/Notion”源链接，系统内不手改。
